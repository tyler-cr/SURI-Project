## Summary 

(Subterranean) Seismic Imaging Rover (SIR) is a semi-autonomous robotic research platform designed to map subsurface geological structures using active seismic reflection techniques and convolutional neural networks. This system utilizes low-frequency seismic waves generated by a controlled agitator and captured by a high-sensitivity geophone array.

The system processes the seismic data in real-time with a Raspberry Pi 5, converting it into spectrograms and classifying anomalies and specific instances (sinkholes, pipes, rock formations, liquids, etc.) using a custom convolutional neural network.

### Primary Objectives

1. Autonomous Surveying Navigate a user-defined 2D/3D grid using the ROS2-based UGV platform.

2. Data Aquisition Automate the generation of seismic pulses and the recording of ground vibrations at discrete waypoints

3. Signal Processing Convert raw .wav seismic data into spectrograms for feature extraction

4. Machine Learning Train and deploy CNN models to classify subsurface composition based on acoustic signatures

5. Dual-Rover Coordination Investigate multi-agent swarm logic for large-scale survey coverage

## Technical Implementation Details

### Communication Protocol

The Raspberry Pi communicates with the ESP32 via a JSON packet structure.

#### Movement Command

```

{

    "T": 1,     // T is type of instruction. 

                //Here, 1 is setting a target linear velocity for left and right wheels

    "L": 0.5,

    "R": 0.5 

}

```

#### Seismic Trigger Command

```

{

    "T": 20,        //Custom Trigger Agitator Command

    "Duration": 500 //Optional Duration Param

}

```

## ML Pipline

Utilizing custom CNN architecture trained on spectrogram images derived from seismic data as .wav files.

1. Data Collection Raw .wav files recorded at grid waypoints.

2. Preprocessing Generate spectrograms, augment data, normalize, and resize

3. Training Custom-made model with spectrogram input (244, 244, 1) np array with output classes of Void, Pipe, Rock, and Liquid.

## Safety and Limitations

- Current Status Simulation Phase. Physical deployment requires calibration of the agitator force and geophone sensitivity

- Power Ensure battery levels are monitored, as agitation and geophones may drain batteries quickly.

- Terain 6-wheel chassis, while designed for rough terrain, may have instability at extreme slopes (>30°)
